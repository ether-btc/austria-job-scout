### Subagent 1: fetcher.py Review

#### 1. Random Delays & Long-Tail Distribution (lines 291-298)
```python
def _sleep_like_human() -> None:
    delay = random.uniform(config.DELAY_MIN_S, config.DELAY_MAX_S)
    time.sleep(delay)
    if random.random() < config.LONG_PAUSE_PROB:  # 15% chance
        long_pause = random.uniform(config.LONG_PAUSE_MIN_S, config.LONG_PAUSE_MAX_S)
        logger.info("long-pause %ds (human-like)", int(long_pause))
        time.sleep(long_pause)
```

### Subagent 2: indexer.py Review

**Status:** ❌ 2 CRITICAL ISSUES FOUND (FIXED)

#### Issue 1: Try-Except Pattern (lines 138-141)
**Severity:** Critical
**Finding:** Fallback model failure unhandled

**Before (BROKEN):**
```python
try:
    self.embedding_model = SentenceTransformer("intfloat/e5-small-v2")
except Exception:
    self.embedding_model = SentenceTransformer("paraphrase-multilingual-MiniLM-L12-v2")
```

**Problem:** If both models fail to load, `self.embedding_model` is None, causing runtime error on first `.encode()` call.

**After (FIXED):**
```python
try:
    self.embedding_model = SentenceTransformer("intfloat/e5-small-v2")
except Exception as e:
    print(f"Failed to load e5-small-v2: {e}. Trying multilingual MiniLM...")
    try:
        self.embedding_model = SentenceTransformer("paraphrase-multilingual-MiniLM-L12-v2")
    except Exception as e2:
        print(f"Failed to load multilingual MiniLM: {e2}. Falling back to TF-IDF")
        self.use_ml = False
        self.tfidf = SimpleEmbedding()
```

#### Issue 2: Silent Failures
**Severity:** Major
**Finding:** No logging on ML model failures

**Fix Applied:**
- Added error messages for both model loading attempts
- Gracefully fall back to TF-IDF if both models fail
- Consistent with existing ImportError handler
